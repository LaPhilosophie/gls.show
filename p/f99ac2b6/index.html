using vscode work space

描述了如何使用Visual Studio Code 工作区

  • 点击start，这样就会启动挑战
  • 点击网站最上方的workspace，将会启动一个网页版vscode
  • 网页版vsc可以使用ctrl+shift+c复制

using the desktop gui

  • 点击start
  • 之后点击上方的desktop，启动远程虚拟机

这地方比较sb的是从网页版虚拟机中复制东西。你在虚拟机复制之后，左侧会有一个剪贴板，再复制一次才能将内容复制到本机，这里它相当于一个中专作用

  1. Click on the ?? button on the left edge of the Desktop.
  2. Click on the Clipboard button. The Dojo’s clipboard will appear!
  3. Overwrite the contents of the dojo’s clipboard with the contents of your
    system’s clipboard (for example, Ctrl-A followed by Ctrl-V).
  4. Now the contents are in the dojo’s clipboard!
  5. Paste the contents into the terminal (Right Click->Paste).

pasting into desktop

同上，教你怎么用剪贴板

the flag file

讲述了flag file：您正在以 hacker 用户身份执行，而 /flag 只能由 root 用户读取，因此您无法访问它。在之前的挑战中，挑战程序本身（例如 /challenge/solve ）以 root 用户身份运行（因此可以读取标志），读取此文件并打印其内容内容

using practice mode

 Practice 模式。这将授予您挑战容器中的管理权限（通过 sudo ），使您能够更强大地调试解决方案并以其他方式尝试挑战。您无法使用它来实际解决挑战，因为在练习模式下， /flag 文件会被替换为无法兑换积分的练习标志。

先进去practice，得到secret。然后进去start，把secret发给终端得到flag

persistent home directories - one

您在道场中的主目录在挑战之间保持不变。这意味着当您开始新的挑战时，您保存在其中的所有文件仍然会在那里。您可以使用它在模块中的多个挑战中建立笔记，参考旧的解决方案，或者针对非练习模式中的挑战重新运行在练习模式中完善的解决方案。

这个挑战和下一个挑战将通过让您在主目录中写入一个文件来阐明要点。发起此挑战并按照其说明进行操作！

persistent home directories - two

现在您已经在主目录中创建了一个文件，本次挑战将使用它！发起此挑战并按照其说明进行操作！

Congratulations! The file that you created in the last challenge is still
present in your home directory! Please keep in mind that, though the contents
of /home/hacker persist, other areas (such as /tmp, /challenge, and so on) do
not.
